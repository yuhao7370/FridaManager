# Frida Manager

一个用于在 Android 设备上管理 `frida-server` 的原生应用。

它主要解决这几件事：

- 在手机上导入或下载多个 `frida-server` 版本
- 按设备 ABI 自动匹配可用资源
- 一键切换当前活动版本
- 直接启动、停止、重启 `frida-server`
- 查看控制器日志、`stdout`、`stderr`
- 配置默认 host、port 和 GitHub API 镜像

## 适用场景

如果你经常在手机上调试、逆向、切换 Frida 版本，这个应用可以把原本靠手动复制文件、`chmod`、`su` 启动、看日志的流程集中到一个界面里。

## 运行前提

使用这个应用前，建议先确认下面几点：

- 设备已 root，并且应用能够拿到可用的 root shell
- Android 版本不低于 8.0（`minSdk = 26`）
- 设备能联网，用于拉取 GitHub Release 或镜像源
- 你准备使用的 `frida-server` 与设备 ABI 匹配

说明：

- 默认监听地址是 `127.0.0.1`
- 默认端口是 `27042`

## 功能概览

### 1. 主页

- 查看当前运行状态
- 查看当前活动版本、PID、host、port、ABI、最近错误
- 启动、停止、重启 `frida-server`

### 2. 版本管理

- 导入本地 `frida-server` 文件
- 刷新远程 Frida 发布列表
- 直接输入精确版本号快速下载
- 管理已安装版本
- 切换当前活动版本
- 删除本地版本
- 查看下载队列和下载进度

### 3. 日志

- 查看控制器日志
- 查看 `frida-server` 标准输出
- 查看 `frida-server` 错误输出
- 手动刷新、清空、复制、导出日志

### 4. 设置

- 设置默认 host / port
- 设置日志保留大小
- 配置 GitHub API 基础地址
- 切换主题和语言

## 怎么用

### 首次使用

1. 安装并打开应用。
2. 确保应用已经获得 root 权限。
3. 进入“版本”页，先准备一个可用版本：
   - 可以点“导入文件”导入本地 `frida-server`
   - 也可以点“刷新远程”，然后从远程列表下载
4. 导入或下载完成后，确认有一个版本处于当前活动状态。
5. 回到“主页”，点击“启动”即可运行当前活动版本的 `frida-server`。

### 导入本地版本

支持两类文件：

- 已解压的 `frida-server` 可执行文件
- `.xz` 压缩包

建议文件名尽量带上版本号，例如：

- `frida-server-17.8.0-android-arm64.xz`
- `frida-server`

如果文件名里能识别出版本号，应用会自动使用该版本号；识别不出来时，会生成一个导入版本名。

### 从远程下载版本

1. 打开“版本”页
2. 点击“刷新远程”
3. 应用会根据设备 ABI 过滤可用资源
4. 选择需要的版本并下载

你也可以直接在“按版本下载”里输入精确版本号，例如：

```text
17.8.0
v17.8.0
```

应用会在当前远程列表里查找这个版本，然后按设备 ABI 下载匹配资源。

### 切换版本

在“已安装版本”列表里点击“切换”即可。

注意：

- 如果当前 `frida-server` 正在运行，应用会先停止当前进程，再切换活动版本，然后按当前设置的 host / port 重新启动
- 删除正在运行的活动版本前，需要先停止 `frida-server`

### 启动和停止

在“主页”里可以直接执行：

- 启动
- 停止
- 重启

启动时会使用“设置”页中的默认 host 和 port。

默认情况下：

- host: `127.0.0.1`
- port: `27042`

如果你需要局域网访问，可以把 host 改成更合适的监听地址，但这会改变暴露面，建议自行评估风险。

### 查看日志

在“日志”页可以查看三类日志：

- 控制器日志
- `frida-server` 标准输出
- `frida-server` 错误输出

同时支持：

- 自动刷新
- 手动刷新
- 清空
- 复制
- 导出

## 远程源配置

默认 GitHub API 地址为：

```text
https://api.github.com
```

应用会基于这个地址请求 Frida Release 列表，所以如果你使用镜像，填写的应当是“GitHub API 根地址”或兼容地址，而不是某个具体 release 页面。

如果远程请求失败：

- 应用会尽量显示本地缓存的发布列表
- 超时或 `504` 时，可以在“设置”里切换镜像 API 地址后再试

## 本地数据

应用会在私有目录下维护自己的 Frida 文件结构，主要包括：

- 已安装版本
- 运行状态
- 下载缓存
- 日志文件

另外，应用也会尝试把版本备份到外部应用目录，用于后续恢复。

## 构建与安装

### 环境要求

- Android Studio 最新稳定版即可
- JDK 17
- Android SDK 35

### 本地构建

```bash
./gradlew assembleDebug
```

生成的调试包可用于本地安装测试。

如果你要构建 release：

```bash
./gradlew assembleRelease
```
